# Proposal: 论文核心功能补全实现

## Summary
根据毕业论文描述的功能与实际代码对比分析，补全10项未实现或不完整的核心功能，确保答辩时每个论文描述的功能都有可运行的代码支撑。

## Motivation
毕业论文中描述了4项核心创新和多项系统功能，但代码审计发现：
- RAG检索管道与聊天流程完全断开
- 动态阈值机制（核心创新#2）无任何代码
- 增强型上下文（核心创新#3）存在但未接入
- SSE流式输出（论文5.4节）未实现
- Milvus向量数据库有配置但无连接代码
- 前端缺少Markdown渲染、来源标注、ECharts图表
- 知识库前端依赖mock数据，不支持文件上传

答辩时若被问及这些功能的实现细节，需要有可运行、可演示的代码。

## Changes
按优先级分3个阶段实现：

### Phase 1: P0 核心创新功能（答辩必备）
1. RAG管道接入聊天流程 - 让AI对话真正使用知识库检索
2. 动态阈值机制 - 按查询意图设置差异化阈值
3. 增强型上下文接入 - 在AI回复中展示来源标注
4. SSE流式输出 - 实现真正的token级流式响应

### Phase 2: P1 关键功能（完整闭环）
5. Milvus向量数据库接入 - 替换内存HashMap，持久化向量
6. 智能路由外部服务调用 - 实际调用产品/订单服务API
7. Markdown渲染 + 来源标注UI - 前端完整展示

### Phase 3: P2 体验增强（锦上添花）
8. ECharts知识库统计图表
9. PDF/TXT文件上传解析
10. 对话摘要压缩机制

## Impact
- Phase 1 完成后：论文4个核心创新全部有代码支撑，可完整演示
- Phase 2 完成后：系统形成完整闭环，从知识库→检索→对话→展示全链路可用
- Phase 3 完成后：系统体验接近论文描述的理想状态
